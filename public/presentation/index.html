The hook

The customer was told it worked.
Did it actually work?

We close the gap between confident agent language and what backend tools really did.

Problem

Success theater in voice AI

  • Calls sound resolved — customer hangs up satisfied
  • Tool traces show 503s, no-ops, or silent failures
  • Support teams lack a public audit trail of who knew what
  • No structured way to contest theories across conversation vs execution

Solution

Investigate in public

Incident Room turns voice failures into structured evidence, recruits specialists, and streams a live investigation — with Band as the collaboration layer.

Investigation flow

Three evidence layers

Layer 1 · Conversation

Transcript, customer belief, premature closure signals

Layer 2 · Execution

Tool calls, HTTP errors, side effects, noop detection

Layer 3 · Context

CRM, chat history, Band room trace, platform metadata

Output

Surviving explanation · fix target · PDF incident report

Built to ship

Production-ready stack

  • Next.js 14 · SSE investigation stream · Vercel deploy
  • MongoDB · chats, failures, incidents, CRM — no runtime JSON files
  • Band API · rooms, agents, visible chat + event fallback
  • Graceful degradation · local room IDs if Band limit hit
